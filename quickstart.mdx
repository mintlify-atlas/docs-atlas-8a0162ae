This guide walks you through setting up Echo's backend and mobile app, connecting your first toolkit, and executing your first automated action.

## Prerequisites

Before you begin, ensure you have:

- **Node.js** v18 or higher
- **pnpm** package manager (`npm install -g pnpm`)
- A [Supabase](https://supabase.com/docs/guides/getting-started/quickstarts/expo-react-native) project (free tier works)
- A [Composio](https://docs.composio.dev/docs/quickstart) account for tool integrations
- A [Supermemory](https://supermemory.ai/docs/quickstart) account for memory management
- An AI API key: [Gemini](https://ai.google.dev/gemini-api/docs/api-key), OpenAI, or Anthropic
- [Expo Go](https://expo.dev/client) app on your mobile device (iOS or Android)

<Warning>
You'll need your LAN IP address to test on a physical device. Run `ipconfig` (Windows) or `ifconfig` (Mac/Linux) to find it.
</Warning>

## Backend setup

<Steps>
  <Step title="Clone and navigate to backend">
    ```bash
    cd backend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file:
    
    ```bash
    cp .env.example .env
    ```
    
    Fill in the required values in `.env`:
    
    ```bash .env
    PORT=8000
    
    # Supabase (get from your project settings)
    SUPABASE_URL=https://your-project.supabase.co
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
    DATABASE_URL=postgresql://postgres:[password]@db.[project-ref].supabase.co:5432/postgres
    
    # AI Model Provider (choose one)
    AI_MODEL_PROVIDER=google  # options: openai, google, anthropic
    GOOGLE_GENERATIVE_AI_API_KEY=your-gemini-key
    # OPENAI_API_KEY=your-openai-key
    # ANTHROPIC_API_KEY=your-anthropic-key
    
    # Composio
    COMPOSIO_API_KEY=your-composio-api-key
    COMPOSIO_WEBHOOK_SECRET=your-webhook-secret
    COMPOSIO_CALLBACK_URL=http://<YOUR_LAN_IP>:8000/api/toolkits/oauth/callback
    
    # Supermemory
    SUPERMEMORY_API_KEY=your-supermemory-api-key
    
    # Frontend URL (for OAuth redirects)
    FRONTEND_URL=http://localhost:5173
    ```
  </Step>

  <Step title="Run database migrations">
    ```bash
    pnpm db:migrate
    ```
    
    This creates the required database tables using Drizzle ORM:
    - `profiles` - User profiles and preferences
    - `action_logs` - Audit trail of executed actions
    - `items` - User's saved Thoughts
    - `user_push_tokens` - Device tokens for notifications
  </Step>

  <Step title="Start the backend server">
    ```bash
    pnpm start
    ```
    
    The server will start on `http://localhost:8000`. You should see:
    
    ```
    Server running on port 8000
    ```
  </Step>
</Steps>

<Info>
The backend exposes REST APIs at `/api/*` for profiles, action logs, toolkits, items, triggers, and push tokens.
</Info>

## Frontend setup

<Steps>
  <Step title="Navigate to frontend directory">
    ```bash
    cd echo
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file:
    
    ```bash
    cp .env.example .env
    ```
    
    Fill in your values:
    
    ```bash .env
    # Supabase
    EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
    EXPO_PUBLIC_SUPABASE_KEY=your-anon-key
    
    # OAuth Redirect (deep link for mobile)
    EXPO_PUBLIC_OAUTH_REDIRECT_URL=echo://auth/callback
    
    # Backend API URL (use your LAN IP for physical device testing)
    EXPO_PUBLIC_API_URL=http://<YOUR_LAN_IP>:8000
    ```
    
    <Note>
    If testing with Expo Go in the simulator, you can use `http://localhost:8000` for the API URL.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    npx expo start
    ```
    
    You'll see a QR code in the terminal:
    
    ```
    › Metro waiting on exp://192.168.1.100:8081
    › Scan the QR code above with Expo Go (Android) or the Camera app (iOS)
    ```
  </Step>

  <Step title="Open on your device">
    - **iOS**: Open the Camera app and scan the QR code
    - **Android**: Open the Expo Go app and scan the QR code
    
    The app will load on your device.
  </Step>
</Steps>

## Your first action

<Steps>
  <Step title="Sign up and complete onboarding">
    1. Open the Echo app on your device
    2. Tap **Get Started** on the intro screen
    3. Sign up with email and password via Supabase Auth
    4. Answer the onboarding questions:
       - **Persona**: Select what describes your day (e.g., "Meetings & coordination")
       - **Work location**: Where your work lives (e.g., "Cloud apps & tools")
       - **Automation posture**: Choose **"Ask me first"** to approve actions before execution
       - **Notifications**: Choose **"All notifications"** to stay informed
  </Step>

  <Step title="Connect your first toolkit">
    After onboarding, you'll see recommended toolkits. Let's connect Google Calendar:
    
    1. Tap **Google Calendar** from the recommendations (or navigate to Toolkits tab)
    2. Review the requested OAuth scopes (calendar read, calendar write, etc.)
    3. Tap **Connect** to initiate OAuth flow
    4. Sign in with your Google account and grant permissions
    5. You'll be redirected back to Echo
    
    <Info>
    Echo uses Composio to manage OAuth flows and toolkit connections. All credentials are stored securely.
    </Info>
  </Step>

  <Step title="Add a Thought">
    Give Echo some context about your preferences:
    
    1. Navigate to the **Thoughts** tab
    2. Tap the **+** button
    3. Add a note like:
       ```
       I prefer morning meetings between 9-11 AM. 
       I always need 10 minutes of prep time before calls.
       ```
    4. Tap **Save**
    
    Echo will use this context when proposing calendar-related actions.
  </Step>

  <Step title="Trigger an action">
    Create a calendar event to trigger Echo:
    
    1. Open Google Calendar (web or mobile)
    2. Create a new event for tomorrow at 2 PM titled "Team Sync"
    3. Add yourself as an attendee
    4. Save the event
    
    Within seconds, Echo will:
    - Receive a trigger webhook from Composio
    - Analyze the event using your connected toolkits and Thoughts
    - Propose an action (e.g., "Check availability and RSVP")
    - Send you a push notification
  </Step>

  <Step title="Review and approve">
    1. Tap the notification to open Echo
    2. Review the proposed action on the **Home** tab
    3. See Echo's reasoning and proposed steps:
       ```
       Thought: User has a new calendar event
       Toolkit: Checked calendar for conflicts
       Thought: No conflicts found
       Proposal: RSVP as "Yes" and block focus time before meeting
       ```
    4. Tap **Approve** to execute, or add feedback to modify
    
    <Tip>
    Try responding with "Also add a reminder 10 minutes before" to see multi-turn conversation in action.
    </Tip>
  </Step>

  <Step title="Watch Echo execute">
    Once approved, Echo will:
    - Update your RSVP status in Google Calendar
    - Create a focus time block before the meeting
    - Report execution results
    - Update the action status to "Completed"
    
    You can view the full execution trace in the action detail screen.
  </Step>
</Steps>

## Testing webhooks on physical devices

For production-like testing with real webhook delivery:

<Steps>
  <Step title="Expose your backend with ngrok">
    ```bash
    ngrok http 8000
    ```
    
    Copy the HTTPS URL (e.g., `https://abc123.ngrok.io`).
  </Step>

  <Step title="Update Composio webhook URL">
    1. Go to the [Composio Dashboard](https://app.composio.dev)
    2. Navigate to **Webhooks** settings
    3. Update the webhook URL to `https://abc123.ngrok.io/api/triggers/webhook`
    4. Save the changes
  </Step>

  <Step title="Test on physical device">
    Webhooks will now be delivered to your local backend through the ngrok tunnel. This only works with a development build, not Expo Go.
  </Step>
</Steps>

<Warning>
ngrok URLs change each time you restart. Consider using a paid ngrok plan with a fixed domain for persistent testing.
</Warning>
