---
title: Unsubscribe from Trigger
description: Disable or permanently delete a trigger subscription to stop receiving webhook notifications
---

## DELETE /api/triggers/:triggerId

Disable or delete a trigger subscription. The trigger will stop sending webhook notifications immediately.

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Path Parameters

<ParamField path="triggerId" type="string" required>
  The unique identifier of the trigger instance to disable/delete
</ParamField>

### Query Parameters

<ParamField query="permanent" type="boolean" default="false">
  If `true`, permanently delete the trigger. If `false` (default), only disable it (can be re-enabled later)
</ParamField>

### Response

**204 No Content** - Trigger successfully disabled/deleted with no response body

<ResponseField name="status" type="string">
  Only returned if not permanently deleted: Status of the operation (`disabled`)
</ResponseField>

<ResponseField name="triggerId" type="string">
  Only returned if not permanently deleted: The ID of the disabled trigger
</ResponseField>

### Example Request

```bash
# Disable a trigger (can be re-enabled later)
curl -X DELETE https://api.echo.app/api/triggers/trigger_xyz789 \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```bash
# Permanently delete a trigger
curl -X DELETE "https://api.echo.app/api/triggers/trigger_xyz789?permanent=true" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

### Example Response

**Disable (default):**
```json
{
  "status": "disabled",
  "triggerId": "trigger_xyz789"
}
```

**Permanent delete:**
```
204 No Content
```

### Error Responses

<ResponseField name="error" type="string">
  Error type identifier
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable error message
</ResponseField>

<Expandable title="Common error responses">
  **401 Unauthorized** - Invalid or missing authentication token
  ```json
  {
    "error": "Unauthorized",
    "message": "Invalid authentication token"
  }
  ```
  
  **404 Not Found** - Trigger not found or not authorized
  ```json
  {
    "error": "Not Found",
    "message": "Trigger not found or you are not authorized"
  }
  ```
  
  **500 Internal Server Error** - Server error during disable/delete
  ```json
  {
    "error": "Internal Server Error",
    "message": "Failed to disable trigger"
  }
  ```
</Expandable>

## PATCH /api/triggers/:triggerId/enable

Re-enable a previously disabled trigger. The trigger will resume sending webhook notifications.

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Path Parameters

<ParamField path="triggerId" type="string" required>
  The unique identifier of the disabled trigger instance to re-enable
</ParamField>

### Response

<ResponseField name="status" type="string">
  Status of the operation (`active`)
</ResponseField>

<ResponseField name="triggerId" type="string">
  The ID of the re-enabled trigger
</ResponseField>

<ResponseField name="enabledAt" type="string">
  ISO 8601 timestamp when the trigger was re-enabled
</ResponseField>

### Example Request

```bash
curl -X PATCH https://api.echo.app/api/triggers/trigger_xyz789/enable \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

### Example Response

```json
{
  "status": "active",
  "triggerId": "trigger_xyz789",
  "enabledAt": "2025-01-16T14:20:00Z"
}
```

### Error Responses

<Expandable title="Common error responses">
  **401 Unauthorized** - Invalid or missing authentication token
  ```json
  {
    "error": "Unauthorized",
    "message": "Invalid authentication token"
  }
  ```
  
  **404 Not Found** - Trigger not found or already enabled
  ```json
  {
    "error": "Not Found",
    "message": "Trigger not found, already enabled, or you are not authorized"
  }
  ```
  
  **500 Internal Server Error** - Server error during enable
  ```json
  {
    "error": "Internal Server Error",
    "message": "Failed to enable trigger"
  }
  ```
</Expandable>

### Notes

- Disabling a trigger (default) allows you to re-enable it later without reconfiguring
- Permanently deleting a trigger requires creating a new subscription if you want to re-enable the same event
- When a connected account is disconnected, all associated triggers are automatically disabled
- Triggers may be automatically disabled when a superseding toolkit is connected (e.g., Google Docs triggers when Google Drive connects)
- Re-enabling a trigger does not recreate historical events - only new events after enablement will be processed
