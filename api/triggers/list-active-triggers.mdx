## GET /api/triggers

List all trigger instances (subscriptions) with optional filtering by connected account, trigger type, or status.

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Query Parameters

<ParamField query="connectedAccountIds" type="string">
  Comma-separated list of connected account IDs to filter by
  
  Example: `conn_abc123,conn_def456`
</ParamField>

<ParamField query="triggerNames" type="string">
  Comma-separated list of trigger names to filter by
  
  Example: `GITHUB_COMMIT_EVENT,GMAIL_NEW_GMAIL_MESSAGE`
</ParamField>

<ParamField query="showDisabled" type="boolean" default="false">
  Include disabled trigger instances in the response
  
  Set to `true` to include triggers that have been disabled
</ParamField>

<ParamField query="limit" type="number" default="50">
  Maximum number of results to return per page
  
  Range: 1-100
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for pagination (1-indexed)
</ParamField>

### Response

<ResponseField name="items" type="array">
  Array of trigger instance objects
  
  <Expandable title="Trigger instance properties">
    <ResponseField name="id" type="string">
      Unique identifier for the trigger instance
    </ResponseField>
    
    <ResponseField name="triggerName" type="string">
      The trigger type identifier (e.g., `GITHUB_COMMIT_EVENT`)
    </ResponseField>
    
    <ResponseField name="connectedAccountId" type="string">
      The connected account ID this trigger is attached to
    </ResponseField>
    
    <ResponseField name="triggerConfig" type="object">
      Configuration object for this trigger instance
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp when the trigger was created
    </ResponseField>
    
    <ResponseField name="disabledAt" type="string | null">
      ISO 8601 timestamp when the trigger was disabled, or null if active
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Current status: `active`, `disabled`, or `error`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">
  Total number of trigger instances matching the filter criteria
</ResponseField>

<ResponseField name="page" type="number">
  Current page number
</ResponseField>

<ResponseField name="limit" type="number">
  Number of results per page
</ResponseField>

### Example Request

```bash
# List all active triggers
curl -X GET "https://api.echo.app/api/triggers" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```bash
# List triggers for specific connected accounts, including disabled ones
curl -X GET "https://api.echo.app/api/triggers?connectedAccountIds=conn_abc123,conn_def456&showDisabled=true" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```bash
# List only GitHub commit triggers
curl -X GET "https://api.echo.app/api/triggers?triggerNames=GITHUB_COMMIT_EVENT&limit=10" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

### Notes

- Only triggers for connected accounts owned by the authenticated user are returned
- Disabled triggers are excluded by default - use `showDisabled=true` to include them
- Triggers may be automatically disabled when a superseding toolkit is connected (e.g., Google Docs triggers are disabled when Google Drive is connected)
- The response includes pagination metadata to help navigate large result sets
