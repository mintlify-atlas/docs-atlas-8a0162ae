---
title: Subscribe to Trigger
description: Subscribe to a trigger for a connected toolkit account to receive webhook notifications when events occur
---

## POST /api/triggers/subscribe

Create a new trigger subscription for a connected account. When the specified event occurs, Echo will receive a webhook notification and process it according to your automation settings.

### Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Request Body

<ParamField body="userId" type="string" required>
  The unique identifier of the user subscribing to the trigger
</ParamField>

<ParamField body="triggerSlug" type="string" required>
  The identifier for the trigger type (e.g., `GITHUB_COMMIT_EVENT`, `GMAIL_NEW_GMAIL_MESSAGE`)
</ParamField>

<ParamField body="connectedAccountId" type="string" required>
  The ID of the connected account (toolkit connection) to attach the trigger to
</ParamField>

<ParamField body="triggerConfig" type="object" required>
  Configuration object specific to the trigger type
  
  <Expandable title="Example configurations">
    **GitHub commit trigger:**
    ```json
    {
      "owner": "username",
      "repo": "repository-name",
      "branch": "main"
    }
    ```
    
    **Gmail new message trigger:**
    ```json
    {
      "label": "INBOX",
      "from": "specific-sender@example.com"
    }
    ```
    
    **Google Calendar event trigger:**
    ```json
    {
      "calendarId": "primary"
    }
    ```
  </Expandable>
</ParamField>

### Response

<ResponseField name="status" type="string">
  Status of the trigger creation (e.g., `success`)
</ResponseField>

<ResponseField name="triggerId" type="string">
  Unique identifier for the created trigger instance
</ResponseField>

<ResponseField name="triggerName" type="string">
  Human-readable name of the trigger
</ResponseField>

<ResponseField name="connectedAccountId" type="string">
  The connected account ID this trigger is attached to
</ResponseField>

### Example Request

```bash
curl -X POST https://api.echo.app/api/triggers/subscribe \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "userId": "user_12345",
    "triggerSlug": "GITHUB_COMMIT_EVENT",
    "connectedAccountId": "conn_abc123",
    "triggerConfig": {
      "owner": "echo-ai",
      "repo": "echo-app",
      "branch": "main"
    }
  }'
```

### Example Response

```json
{
  "status": "success",
  "triggerId": "trigger_xyz789",
  "triggerName": "GITHUB_COMMIT_EVENT",
  "connectedAccountId": "conn_abc123"
}
```

### Error Responses

<ResponseField name="error" type="string">
  Error type identifier
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable error message
</ResponseField>

<Expandable title="Common error responses">
  **400 Bad Request** - Invalid trigger configuration
  ```json
  {
    "error": "Bad Request",
    "message": "Invalid trigger configuration",
    "details": "Missing required field: owner"
  }
  ```
  
  **401 Unauthorized** - Invalid or missing authentication token
  ```json
  {
    "error": "Unauthorized",
    "message": "Invalid authentication token"
  }
  ```
  
  **404 Not Found** - Connected account not found
  ```json
  {
    "error": "Not Found",
    "message": "Connected account not found or you are not authorized"
  }
  ```
  
  **500 Internal Server Error** - Server error during trigger creation
  ```json
  {
    "error": "Internal Server Error",
    "message": "Failed to create trigger subscription"
  }
  ```
</Expandable>

### Notes

- Triggers are automatically enabled upon creation unless the toolkit is superseded by another connected toolkit (e.g., Google Docs triggers are disabled if Google Drive is also connected)
- Multiple triggers can be created for the same connected account with different configurations
- Trigger events are sent to the webhook endpoint at `/api/triggers/webhook`
- Scopes required for each trigger type depend on the toolkit - ensure the connected account has the necessary OAuth scopes enabled
