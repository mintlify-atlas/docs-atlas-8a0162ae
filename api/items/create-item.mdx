---
title: "Create Item"
description: "Create a new thought item with URL, notes, and tags"
---

## Endpoint

<ParamField path="method" type="string">
  POST
</ParamField>

<ParamField path="url" type="string">
  `/api/items`
</ParamField>

## Authentication

This endpoint requires authentication. Include a valid Bearer token in the Authorization header.

```bash
Authorization: Bearer <your_access_token>
```

## Request Body

<ParamField body="url" type="string" required>
  The URL of the item to save. Must be a valid URL format.
</ParamField>

<ParamField body="title" type="string">
  Optional title for the item. Can be null.
</ParamField>

<ParamField body="note" type="string">
  Optional note or description for the item. Can be null.
</ParamField>

<ParamField body="tags" type="array">
  Optional array of tag strings to categorize the item. Defaults to empty array.
</ParamField>

<ParamField body="og_image" type="string">
  Optional Open Graph image URL. Must be a valid URL format if provided. Can be null.
</ParamField>

<ParamField body="og_description" type="string">
  Optional Open Graph description. Can be null.
</ParamField>

## Response

<ResponseField name="item" type="object">
  The created item object
  
  <Expandable title="item properties">
    <ResponseField name="id" type="string">
      UUID of the created item
    </ResponseField>
    
    <ResponseField name="userId" type="string">
      UUID of the user who owns the item
    </ResponseField>
    
    <ResponseField name="url" type="string">
      The saved URL
    </ResponseField>
    
    <ResponseField name="title" type="string | null">
      Title of the item
    </ResponseField>
    
    <ResponseField name="note" type="string | null">
      Note or description
    </ResponseField>
    
    <ResponseField name="tags" type="array">
      Array of tag strings
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Item status. Default is "active"
    </ResponseField>
    
    <ResponseField name="isRead" type="boolean">
      Whether the item has been read. Default is false
    </ResponseField>
    
    <ResponseField name="ogImage" type="string | null">
      Open Graph image URL
    </ResponseField>
    
    <ResponseField name="ogDescription" type="string | null">
      Open Graph description
    </ResponseField>
    
    <ResponseField name="supermemoryId" type="string">
      ID of the item in Supermemory service
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of creation
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X POST https://api.echo.app/api/items \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/article",
    "title": "Interesting Article",
    "note": "Need to read this later",
    "tags": ["reading", "tech"],
    "og_image": "https://example.com/image.jpg",
    "og_description": "A fascinating article about technology"
  }'
```

## Notes

- Items are automatically synced to Supermemory for AI-powered memory retrieval
- If Supermemory sync fails, the entire operation fails to maintain consistency
- All timestamps are in ISO 8601 format with timezone information
- Tags must be an array of strings; empty array is the default
