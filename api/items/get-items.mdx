---
title: "Get Items"
description: "Retrieve a paginated list of thought items"
---

## Endpoint

<ParamField path="method" type="string">
  GET
</ParamField>

<ParamField path="url" type="string">
  `/api/items`
</ParamField>

## Authentication

This endpoint requires authentication. Include a valid Bearer token in the Authorization header.

```bash
Authorization: Bearer <your_access_token>
```

## Query Parameters

<ParamField query="status" type="string">
  Filter items by status. Possible values: `active`, `archived`
</ParamField>

<ParamField query="page" type="number">
  Page number for pagination. Minimum: 1. Default: 1
</ParamField>

<ParamField query="limit" type="number">
  Number of items per page. Minimum: 1, Maximum: 100. Default: 20
</ParamField>

## Response

<ResponseField name="items" type="array">
  Array of item objects
  
  <Expandable title="item properties">
    <ResponseField name="id" type="string">
      UUID of the item
    </ResponseField>
    
    <ResponseField name="userId" type="string">
      UUID of the user who owns the item
    </ResponseField>
    
    <ResponseField name="url" type="string">
      The saved URL
    </ResponseField>
    
    <ResponseField name="title" type="string | null">
      Title of the item
    </ResponseField>
    
    <ResponseField name="note" type="string | null">
      Note or description
    </ResponseField>
    
    <ResponseField name="tags" type="array">
      Array of tag strings
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Item status: "active" or "archived"
    </ResponseField>
    
    <ResponseField name="isRead" type="boolean">
      Whether the item has been read
    </ResponseField>
    
    <ResponseField name="ogImage" type="string | null">
      Open Graph image URL
    </ResponseField>
    
    <ResponseField name="ogDescription" type="string | null">
      Open Graph description
    </ResponseField>
    
    <ResponseField name="supermemoryId" type="string | null">
      ID of the item in Supermemory service
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of creation
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata
  
  <Expandable title="pagination properties">
    <ResponseField name="page" type="number">
      Current page number
    </ResponseField>
    
    <ResponseField name="limit" type="number">
      Items per page
    </ResponseField>
    
    <ResponseField name="total" type="number">
      Total number of items
    </ResponseField>
    
    <ResponseField name="totalPages" type="number">
      Total number of pages
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X GET "https://api.echo.app/api/items?status=active&page=1&limit=20" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Notes

- Items are returned in descending order by creation date (newest first)
- Only items belonging to the authenticated user are returned
- Pagination parameters are validated: page >= 1, limit between 1 and 100
- If no items match the filters, an empty array is returned with pagination metadata
