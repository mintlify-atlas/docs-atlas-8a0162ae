---
title: "Create Action Log"
description: "Create a new action log entry to track AI assistant actions"
---

## Endpoint

```
POST /api/action-logs
```

## Authentication

This endpoint requires authentication. Include a valid JWT token in the Authorization header.

```
Authorization: Bearer <token>
```

## Request Body

<ParamField body="requestType" type="string" required>
  Type of request that triggered the action. Must be one of: `chat`, `trigger`
</ParamField>

<ParamField body="triggerSource" type="string" required>
  Source that triggered the action (e.g., webhook name, scheduler ID)
</ParamField>

<ParamField body="actionName" type="string" required>
  Name or description of the action being logged. Minimum 1 character.
</ParamField>

<ParamField body="executionId" type="string">
  Optional execution ID for tracking related operations
</ParamField>

<ParamField body="toolkitsUsed" type="array">
  Array of toolkit slugs used for this action. Each slug must match format `^[a-z0-9]+$`.
  
  <Expandable title="Validation">
    The API validates that all requested toolkits are enabled and connected for the authenticated user.
    If any toolkit is not enabled, the request will fail with a 400 error.
  </Expandable>
</ParamField>

<ParamField body="request" type="string">
  Original user request text that triggered the action
</ParamField>

<ParamField body="additionalContext" type="string">
  Additional context information for the action
</ParamField>

<ParamField body="metadata" type="object">
  Optional metadata as key-value pairs for storing custom information
</ParamField>

## Response

<ResponseField name="log" type="object">
  The created action log object
  
  <Expandable title="Action Log Properties">
    <ResponseField name="id" type="string">
      Unique UUID identifier for the action log
    </ResponseField>
    
    <ResponseField name="userId" type="string">
      ID of the authenticated user
    </ResponseField>
    
    <ResponseField name="requestType" type="string">
      Type of request: `chat` or `trigger`
    </ResponseField>
    
    <ResponseField name="triggerSource" type="string">
      Source that triggered the action
    </ResponseField>
    
    <ResponseField name="actionName" type="string">
      Name of the action
    </ResponseField>
    
    <ResponseField name="executionId" type="string" nullable>
      Execution ID if provided
    </ResponseField>
    
    <ResponseField name="toolkitsUsed" type="array">
      Array of toolkit slugs used
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Status of the action. Will be `pending` for newly created logs.
    </ResponseField>
    
    <ResponseField name="request" type="string" nullable>
      Original request text
    </ResponseField>
    
    <ResponseField name="additionalContext" type="string" nullable>
      Additional context information
    </ResponseField>
    
    <ResponseField name="metadata" type="object" nullable>
      Custom metadata object
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of creation
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
  </Expandable>
</ResponseField>

## Status Codes

- `201` - Action log created successfully
- `400` - Bad request (validation errors or toolkit not enabled)
- `401` - Unauthorized (missing or invalid authentication)
- `500` - Internal server error

## Example Request

```bash
curl -X POST https://api.echo.ai/api/action-logs \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "requestType": "trigger",
    "triggerSource": "github_webhook",
    "actionName": "Create GitHub issue",
    "toolkitsUsed": ["github"],
    "request": "Create an issue for bug found in production",
    "additionalContext": "Error occurred in user authentication flow",
    "metadata": {
      "priority": "high",
      "repository": "echo-ai/backend"
    }
  }'
```
