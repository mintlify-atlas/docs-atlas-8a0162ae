title: "Update Push Token"
description: "Update an existing Expo push token for the authenticated user"
---

## Endpoint

<ParamField path="method" type="string">
  POST
</ParamField>

<ParamField path="endpoint" type="string">
  /api/push-tokens
</ParamField>

## Authentication

This endpoint requires authentication. Include a valid access token in the Authorization header.

## Overview

This endpoint shares the same implementation as the [Register Push Token](/api/push-tokens/register-token) endpoint. It automatically handles both registration and updates using upsert logic:

- If the push token already exists, it updates the record with new information
- If the push token doesn't exist, it creates a new record
- The `lastSeenAt` timestamp is always updated to track token freshness

## Request Body

<ParamField body="expoPushToken" type="string" required>
  The Expo push token to update. Must be in the format `ExponentPushToken[...]` or `ExpoPushToken[...]`.
</ParamField>

<ParamField body="platform" type="string">
  The updated platform of the device (e.g., "ios", "android").
</ParamField>

<ParamField body="deviceId" type="string">
  The updated unique identifier for the device.
</ParamField>

## Response

<ResponseField name="pushToken" type="object">
  The updated push token object.

  <Expandable title="pushToken properties">
    <ResponseField name="id" type="string">
      Unique identifier for the push token record.
    </ResponseField>

    <ResponseField name="userId" type="string">
      The ID of the user who owns this push token.
    </ResponseField>

    <ResponseField name="expoPushToken" type="string">
      The Expo push token.
    </ResponseField>

    <ResponseField name="platform" type="string" optional>
      The platform of the device.
    </ResponseField>

    <ResponseField name="deviceId" type="string" optional>
      The unique device identifier.
    </ResponseField>

    <ResponseField name="lastSeenAt" type="string">
      Timestamp of when the token was last seen/updated (will be the current time).
    </ResponseField>

    <ResponseField name="createdAt" type="string">
      Timestamp of when the token was first registered (unchanged if updating).
    </ResponseField>

    <ResponseField name="updatedAt" type="string">
      Timestamp of when the token was last updated (will be the current time).
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash
curl -X POST https://api.echo.com/api/push-tokens \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "expoPushToken": "ExponentPushToken[xxxxxxxxxxxxxxxxxxxxxx]",
    "platform": "android",
    "deviceId": "updated-device-uuid"
  }'
```

## Use Cases

Use this endpoint to:

- Update platform information when a device changes operating systems
- Refresh the `lastSeenAt` timestamp to prevent token cleanup
- Update device identifiers when necessary
- Ensure the token remains active in the system

## Automatic Cleanup

When updating a token, the system automatically removes stale tokens for the same user that haven't been seen in 90 days (configurable via `PUSH_TOKEN_STALE_DAYS` environment variable).
