---
title: "Enable Toolkit"
description: "Enable a toolkit connection to allow Echo to use it"
---

## Endpoint

```
PATCH /api/toolkits/:id
```

## Authentication

This endpoint requires authentication. The API automatically verifies that the authenticated user owns the toolkit connection.

## Path Parameters

<ParamField path="id" type="string" required>
  The connected account ID (not the auth config ID)
</ParamField>

## Request Body

<ParamField body="enabled" type="boolean" required>
  Set to `true` to enable the toolkit, or `false` to disable it.
  
  When disabled, Echo will not use this toolkit connection for actions or triggers.
</ParamField>

## Response

<ResponseField name="toolkit" type="object">
  The updated toolkit connection object
  
  <Expandable title="ToolkitConnection properties">
    <ResponseField name="id" type="string">
      Connected account ID from Composio
    </ResponseField>
    
    <ResponseField name="slug" type="string">
      Toolkit identifier (e.g., `github`, `gmail`, `slack`)
    </ResponseField>
    
    <ResponseField name="connectionStatus" type="string">
      Current status: `connected`, `disconnected`, `pending`, or `expired`
    </ResponseField>
    
    <ResponseField name="enabled" type="boolean">
      Whether the toolkit is enabled (will reflect the new state)
    </ResponseField>
    
    <ResponseField name="scopes" type="array">
      OAuth scopes granted for this connection
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp when the connection was created
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp when the connection was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

### Enable a toolkit

```bash
curl -X PATCH 'https://api.echo.ai/api/toolkits/acc_abc123xyz' \
  -H 'Cookie: session=your-session-token' \
  -H 'Content-Type: application/json' \
  -d '{
    "enabled": true
  }'
```

### Disable a toolkit

```bash
curl -X PATCH 'https://api.echo.ai/api/toolkits/acc_abc123xyz' \
  -H 'Cookie: session=your-session-token' \
  -H 'Content-Type: application/json' \
  -d '{
    "enabled": false
  }'
```

### Response

```json
{
  "toolkit": {
    "id": "acc_abc123xyz",
    "slug": "github",
    "connectionStatus": "connected",
    "enabled": true,
    "scopes": ["repo", "user"],
    "createdAt": "2026-01-15T10:30:00.000Z",
    "updatedAt": "2026-03-03T09:45:00.000Z"
  }
}
```

## Error Responses

<ResponseField name="error" type="string">
  Error type identifier
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable error message
</ResponseField>

### 400 Bad Request

Returned when the request body is invalid or missing required fields.

```json
{
  "error": "Bad Request",
  "message": "enabled: Expected boolean, received string"
}
```

### 404 Not Found

Returned when the toolkit doesn't exist or the user doesn't have access.

```json
{
  "error": "Not Found",
  "message": "Toolkit not found or you are not authorized"
}
```

### 500 Internal Server Error

```json
{
  "error": "Internal Server Error",
  "message": "Failed to update toolkit"
}
```

## Notes

- Disabling a toolkit does not disconnect it - the OAuth connection remains active
- Re-enabling a toolkit immediately allows Echo to use it again
- This endpoint can also be used to update toolkit scopes by passing a `scopes` array instead of `enabled`, which triggers a re-authentication flow
