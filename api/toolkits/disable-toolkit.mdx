---
title: "Disable Toolkit"
description: "Disable a toolkit connection to prevent Echo from using it"
---

## Endpoint

```
PATCH /api/toolkits/:id
```

## Authentication

This endpoint requires authentication. The API automatically verifies that the authenticated user owns the toolkit connection.

## Path Parameters

<ParamField path="id" type="string" required>
  The connected account ID (not the auth config ID)
</ParamField>

## Request Body

<ParamField body="enabled" type="boolean" required>
  Set to `false` to disable the toolkit.
  
  When disabled, Echo will not use this toolkit connection for actions or triggers, but the OAuth connection remains active.
</ParamField>

## Response

<ResponseField name="toolkit" type="object">
  The updated toolkit connection object
  
  <Expandable title="ToolkitConnection properties">
    <ResponseField name="id" type="string">
      Connected account ID from Composio
    </ResponseField>
    
    <ResponseField name="slug" type="string">
      Toolkit identifier (e.g., `github`, `gmail`, `slack`)
    </ResponseField>
    
    <ResponseField name="connectionStatus" type="string">
      Current status: `connected`, `disconnected`, `pending`, or `expired`
    </ResponseField>
    
    <ResponseField name="enabled" type="boolean">
      Whether the toolkit is enabled (will be `false`)
    </ResponseField>
    
    <ResponseField name="scopes" type="array">
      OAuth scopes granted for this connection
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp when the connection was created
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp when the connection was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

## Example

### Request

```bash
curl -X PATCH 'https://api.echo.ai/api/toolkits/acc_abc123xyz' \
  -H 'Cookie: session=your-session-token' \
  -H 'Content-Type: application/json' \
  -d '{
    "enabled": false
  }'
```

### Response

```json
{
  "toolkit": {
    "id": "acc_abc123xyz",
    "slug": "github",
    "connectionStatus": "connected",
    "enabled": false,
    "scopes": ["repo", "user"],
    "createdAt": "2026-01-15T10:30:00.000Z",
    "updatedAt": "2026-03-03T09:45:00.000Z"
  }
}
```

<ResponseField name="error" type="string">
  Error type identifier


### 404 Not Found

Returned when the toolkit doesn't exist or the user doesn't have access.

```json

  "message": "Toolkit not found or you are not authorized"
}
```

### 500 Internal Server Error

```json
{
  "error": "Internal Server Error",
  "message": "Failed to update toolkit"
}
```

## Disable vs Disconnect

Disabling a toolkit is different from disconnecting it:

- **Disable** (`PATCH /api/toolkits/:id` with `enabled: false`): Prevents Echo from using the toolkit while keeping the OAuth connection active. You can re-enable it at any time without re-authenticating.

- **Disconnect** (`DELETE /api/toolkits/:id`): Completely removes the OAuth connection. You'll need to re-authenticate to use the toolkit again.

Use disable when you want to temporarily prevent Echo from accessing a toolkit without losing the connection.
