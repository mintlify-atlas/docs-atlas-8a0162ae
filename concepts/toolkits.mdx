Toolkits are the integrations that power Echo's ability to perform actions. Each toolkit represents a connected app or service (like Gmail, GitHub, Calendar) that Echo can interact with on your behalf.

## What are toolkits?

A toolkit is a Composio integration that gives Echo access to specific app functionality through OAuth or API key authentication. When you connect a toolkit, you grant Echo permission to:

- Read data from the app (e.g., calendar events, emails)
- Perform actions (e.g., send email, create task, update issue)
- Subscribe to triggers (e.g., new email received, meeting scheduled)

<Info>
Echo uses [Composio](https://composio.dev) to manage toolkit integrations, which provides 150+ pre-built app connections with OAuth handling.
</Info>

## How toolkits work

<Steps>
  <Step title="Browse available toolkits">
    Open the **Toolkits** tab in the mobile app to see all available integrations
  </Step>
  <Step title="Connect via OAuth">
    Tap **Connect** to initiate the OAuth flow and authenticate with the service
  </Step>
  <Step title="Grant permissions">
    Choose which triggers to subscribe to (e.g., "New Email", "Calendar Event Updated")
  </Step>
  <Step title="Enable/disable">
    Toggle toolkits on or off without disconnecting completely
  </Step>
  <Step title="Use in actions">
    Echo automatically uses connected toolkits when proposing and executing actions
  </Step>
</Steps>

## Toolkit lifecycle

Each toolkit connection has a state:

| State | Description |
|-------|-------------|
| **Not connected** | Toolkit available but not authenticated |
| **Connected & enabled** | Active and available for actions |
| **Connected & disabled** | Authenticated but temporarily paused |
| **Expired** | OAuth token expired, needs re-authentication |

## OAuth flow

When you connect a toolkit, Echo handles OAuth securely:

```typescript
// From backend/src/routes/toolkits.ts:127
POST /api/toolkits/:slug/enable
```

1. **Initiate OAuth**: Backend generates OAuth URL for the toolkit
2. **User authentication**: You're redirected to the service to grant permissions
3. **Callback handling**: Service redirects back to Echo with authorization code
4. **Token storage**: Composio stores access/refresh tokens securely
5. **Connection active**: Toolkit is now available for actions

<Warning>
Echo never stores your passwords. All authentication uses OAuth tokens that can be revoked at any time.
</Warning>

## Toolkit permissions

Each toolkit requests specific permissions (scopes) during OAuth:

**Gmail example:**
- `gmail.readonly` - Read email messages
- `gmail.send` - Send emails on your behalf
- `gmail.modify` - Archive, delete, or label messages

**GitHub example:**
- `repo` - Access repository data
- `issues` - Create and update issues
- `pull_requests` - Manage pull requests

<Tip>
You can review and revoke permissions at any time from your account settings in the connected service.
</Tip>

## Using toolkits in actions

When Echo proposes an action, it automatically determines which toolkits are needed:

```json
{
  "action_name": "schedule_meeting_and_prepare_agenda",
  "toolkits_used": ["googlecalendar", "googledrive", "gmail"]
}
```

If a required toolkit isn't connected, Echo will:
1. Notify you which toolkit is needed
2. Provide a link to connect it
3. Pause the action until connection is complete

## Toolkit triggers

Toolkits provide triggers that start automated workflows:

```typescript
// Example trigger configuration
{
  "triggerSlug": "gmail_new_email_received",
  "connectedAccountId": "<your-gmail-connection>",
  "triggerConfig": {
    "labels": ["IMPORTANT"],
    "from": "specific-sender@example.com"
  }
}
```

See [Triggers](/concepts/triggers) for more details.

## Troubleshooting

<Accordion title="Toolkit shows as expired">
OAuth tokens typically expire after 60-90 days. Simply:
1. Tap the toolkit in the app
2. Select **Reconnect**
3. Re-authenticate via OAuth
</Accordion>

<Accordion title="Action failed due to missing permissions">
The toolkit may need additional scopes:
1. Disconnect the toolkit
2. Reconnect and grant all requested permissions
3. Retry the action
</Accordion>

<Accordion title="Toolkit not receiving triggers">
Check webhook configuration:
1. Verify triggers are enabled in toolkit settings
2. Ensure webhook URL is set in Composio dashboard
3. Test with a manual trigger event
</Accordion>
